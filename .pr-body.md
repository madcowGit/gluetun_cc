### Summary

This PR restructures the repository into the standard **Home Assistant custom integration** layout, adds minimal HACS metadata, introduces CI checks (HACS Action + hassfest), and updates the README with instructions for adding this as a **HACS custom repository**, prerequisites, and release guidance.

### Changes

- Move integration files to `custom_components/gluetun_cc/` (required by HACS/HA integration layout). 〔HACS Integration structure docs〕
- Add root-level `hacs.json` with display name for HACS. 〔HACS publish/general - hacs.json〕
- Ensure `manifest.json` includes required keys (`domain`, `name`, `version`, `documentation`, `issue_tracker`, `codeowners`, etc.). 〔HACS Integration requirements〕
- Add CI:
  - **HACS Action** for repo validation. 〔HACS default inclusion docs〕
  - **hassfest** for integration validation. 〔HACS default inclusion docs〕
- Update **README**:
  - HACS custom repository installation steps. 〔HACS custom repositories FAQ〕
  - Requirements (Gluetun HTTP Control Server endpoints).
  - Release workflow for nicer HACS UX and eventual default inclusion.

### Why

HACS expects a known repository structure (`custom_components/<domain>/`) and a `hacs.json` in the repository root so it can install and manage the integration.  
If we decide to pursue **default inclusion** later, HACS requires passing **HACS Action** + **hassfest** and publishing **GitHub releases**.
